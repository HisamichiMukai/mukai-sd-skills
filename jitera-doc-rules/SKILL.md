# Jiteraドキュメント共通ルール

## ドキュメント作成・更新

- 既存ドキュメントを確認してから作成・更新を行う

- 作成・更新前にユーザーへ内容を提示し確認を取る

- フォーマットはMarkdownを基本とし、見出し・箇条書きで構造化する

## ドキュメント管理

ドキュメントの移動・削除は実行前に必ずユーザーの最終確認を取る
